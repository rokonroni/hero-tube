Discuss the scope of var, let, and const:-

var: var declared are function scoped variables. When we declared a variable with var we can use the variable any places we want. If we declared a variable outsite a funcion we can also use the variable in the funcion. And if we declared a variable inside a function we can use the variable outside the funcion . The var variable are hoisted to the top there contaning funcion or the global scope. We can call the variable before initialized and it will give a undefined value.

let: let declared are block scoped variables. When we declared a variable with let we can't use the variable outside a block. Like if we declared a global variable with let we can't use the variable in a function. On the other side if we declared a variable with let in a funcion we can not use the variable outside the function. It work in a boundary or a room which called a block in Programming Language. That's why let variables scoped is block scoped. And the let variables are not hoisted. We can use the variable only when we initialized the variable.

const: const also declared are block scoped variables like let. When we declared a variable with const we cann't be reassigned the initial value of this variable. The const variable is mutable.

Tell us the use cases of null and undefined:-

null:

  • Which variable or property was declared but hasn't been initialized the value we can use 'null' for the variable or property intentionally so the value is empty or no value.
  • We can use null value for an object property to indicate the property is cleared or reset.
  • We can use 'null' to check a specific value or object property exixts or not. And then we can say is it true or false and also is it undefined or not.
  • When we fetch data from API if the functions return 'null' to indicate the abssence of data or an error condition. It's important to check for 'null' when working with this APIs.

    • undefined:

  • When we declared a variable but don't assign a value to the variable, it's automatically initialized with 'undefined'. This can happan when a function paremeter is not provided or when a variable is declared but not assigned a value
  • When we want to access an object property which is doesn't exixts. JavaScript returns 'undefined' the value. So we can check the property exixts in this object or not.
  • When we write any function with no returns it automatically return a 'undefined' by default.

    • What do you mean by REST API?

    REST API: REST API means Representational State Transfer Application Programming Interface. In Rest API, everything is treated as a resource . We can named any pieces of the information.

    REST HTTP Methods: REST uses standard HTTP methods to perform operations on resources. The most commonly used HTTP methods in RESTful APIs are:

  • GET: Used to retrieve data from the server.
  • POST: Used to create a new resource on the server.
  • PUT: Used to update an existing resource or create a new resource if it doesn't exist.
  • DELETE: Used to remove a resource from the server.

    • REST followe a client server architecture and responses from the server can be explicitly marked as cacheable or non cacheable.
